## Commands

Everything runs inside Docker. Source is volume-mounted so hot reload works without rebuilding.

```bash
# First time — build base image (must precede other builds)
docker compose -f docker/docker-compose.yaml build hsm-app-be-base

# Start full stack (server + worker + postgres + redis + minio + consoles)
docker compose -f docker/docker-compose.yaml up

# Start only infra (no app containers)
docker compose -f docker/docker-compose.yaml up hsm-app-db-postgres hsm-app-db-redis hsm-app-db-minio

# Exec into running server container to run ad-hoc commands
docker exec -it hsm-app-be-server sh

# Lint / format (run inside container)
pnpm lint
pnpm lint:fix
pnpm check:fix            # lint + format together

# Tests (run inside container)
pnpm test
pnpm test:watch
pnpm test:cov
pnpm test:e2e
pnpm test -- --testPathPattern=coms.service   # single file

# Build (inside container)
pnpm build:server
pnpm build:worker
```

## Architecture

NestJS monorepo with two apps and five shared libs.

### Apps

| App | Purpose |
|-----|---------|
| `apps/server` | HTTP API. Bootstraps on port 3000 with URI versioning (default v1). Swagger at `/api`. |
| `apps/worker` | BullMQ job processor. No HTTP. Consumes queues from Redis. |

### Libs (`@hsm-lib/*`)

| Lib | What it exports |
|-----|----------------|
| `@hsm-lib/config` | `envs` — frozen, Joi-validated env vars. Import this everywhere instead of `process.env`. |
| `@hsm-lib/database` | Global `DatabaseModule`. Exposes two TypeORM data sources: `hsm-db-postgres` (Postgres) and `hsm-db-oracle` (Oracle). Also contains a codegen CLI that introspects an Oracle DB and emits TypeORM entities. |
| `@hsm-lib/queue` | Global `QueueModule`. BullMQ wired to Redis. Queues: `coms`, `document`, `notification`. 3 attempts, 1 s delay, 2 s backoff. |
| `@hsm-lib/common` | Shared DTOs, interfaces, enums, errors, types. |
| `@hsm-lib/storage` | S3-compatible storage (`StorageService`). Supports path-style (self-hosted) and AWS-native modes via `HSM_STORAGE_S3_FORCE_PATH_STYLE`. |

### Database patterns

- Inject repositories with `@InjectRepository(Entity, Databases.HsmDbPostgres)` or `Databases.HsmDbOracle`.
- `synchronize: true` only in `dev`. Never in production.
- Entity files live in `libs/database/src/entities/modules/<domain>/`. Autogenerated Oracle entities go in `entities/autogenerated/oracle/`.
- To generate entities from an existing Oracle table: `pnpm generator:entity -- --driver=oracle --host=... --user=... --pass=... --db=... --table=... --schema=... --save`.

### Request/response flow

Every response is wrapped by `ResponseInterceptor` into `SuccessResponseDto`. TypeORM exceptions are caught by `TypeOrmExceptionFilter` (in `DatabaseModule`). HTTP errors use `ResponseFilter`. Rate limits: 3/s, 20/10 s, 100/min (ThrottlerGuard global).

Auth uses two JWTs: access token (`JWT_AT_SECRET`) and refresh token (`JWT_RT_SECRET`). Protected routes use `AuthJwtAtGuard` exported from `AuthModule`.

### Tooling

- **Linter/formatter**: Biome (not ESLint/Prettier). Config in `biome.json`. Single quotes, 2-space indent, trailing commas, `noExplicitAny` error.
- **Pre-commit**: Husky + lint-staged.
- **Package manager**: pnpm.
